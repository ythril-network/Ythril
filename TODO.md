
## Features

- [ ] **Public knowledge spaces — open pub/sub networks.** Spaces that represent publicly available knowledge (e.g. documentation, reference data) should support a pub/sub network type that peers can join without notifying the publisher. This removes the invite-accept friction for distributing public knowledge — interested parties simply subscribe. Requires: new network type (`public` or `pubsub`), join-without-approval flow, read-only subscriber role, discovery mechanism (e.g. advertised network ID or join link), and governance rules (publisher controls content, subscribers pull only).
